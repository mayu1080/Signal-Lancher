# Signal Launcher — 現場運用・テスト手順

本書は Signal Launcher を現場で運用する際の手順と、本番前のテスト確認方法をまとめたものです。  
開発・セットアップの詳細は [README.md](./README.md) を参照してください。

---

## 1. 本番前チェックリスト

本番当日またはリハーサル前に、以下を確認してください。

| # | 確認項目 | 方法 |
|---|---------|------|
| 1 | PC 時刻・タイムゾーン | Windows 設定 → 日付と時刻。スケジュール発火の基準になる |
| 2 | 画面モード | 本番中は **ショー** モード、設定変更時のみ **セットアップ** |
| 3 | 設定内容 | **設定** タブ（開発者プロファイル）の JSON、または `show.default.json` |
| 4 | 送信先 IP / ポート | **送信先** タブで実機と一致しているか |
| 5 | MIDI デバイス | **MIDI** タブで **接続 OK** が表示されるか |
| 6 | 仮想 MIDI（Ableton 等） | loopMIDI ポート名が `targets` の `outputDevice` と一致するか |
| 7 | ネットワーク | 送信先 PC・照明コントローラーと同一 LAN / 到達可能か |
| 8 | アプリ起動 | `npm run dev` またはビルド済み exe でウィンドウが開くか |
| 9 | 初期モード | バナーが **DRY RUN** であること。確認後に **本番** へ |
| 10 | 設定バックアップ | **設定** → **エクスポート** で JSON を保存（推奨） |

## 2. 起動手順

### 開発環境での起動

```powershell
cd "c:\Users\m6101\ai_playground\signal lancher"
npm run dev
```

### 起動後の確認（ショーモード・発火タブ）

1. ヘッダーが **ショー** / **現場** になっている（配布時の推奨設定）
2. バナーが **DRY RUN — 送信しません** と表示される
3. **プロジェクト** に `projectName` が表示される
4. 左: **Cue 発火** ボタン、右: **モニター** が並んで表示される
5. MIDI 未接続時は発火タブ上部に警告バナーが出る → **MIDI** タブで **接続 OK** を確認

起動時エラー（設定の致命的欠落）がある場合、ウィンドウが開かないかエラーダイアログが出ます。  
target 参照エラーなどはモニターに `[ERROR]` が出ますが、アプリは起動を続けます。

## 3. 画面構成（日本語 UI）

### ヘッダー

| 項目 | 説明 |
|------|------|
| **ショー** | 発火 + MIDI のみ（現場運用） |
| **セットアップ** | 全タブ表示（設定作業） |
| **現場** | ラベルのみ、技術情報を隠す |
| **開発者** | ID・JSON プレビューを表示 |
| **セットアップウィザード** | 初回設定ガイド |

### タブ（セットアップモード時）

| タブ | 内容 |
|------|------|
| **発火** | ダッシュボード、Cue 発火（2 カラム + モニター）、MIDI 警告 |
| **Cue** | Cue の CRUD、MIDI Learn、並び替え |
| **送信先** | 送信先 CRUD、単体テスト |
| **スケジュール** | スケジュール CRUD、PC 時刻、テスト発火 |
| **設定** | 案件名、再読込、エクスポート/インポート、JSON（開発者のみ） |
| **MIDI** | 接続 OK/NG、デバイス一覧 |

**ショーモード** では **発火** と **MIDI** タブのみ表示されます。

**注意:** ログ本文・triggerId・device 名・config の値は英語/そのまま表示されます（UI ラベルのみ日本語）。

## 3.1 ショーモード vs セットアップモード

| 操作 | ショー | セットアップ |
|------|--------|-------------|
| Cue 発火（クリック） | ✓ | ✓ |
| キー 1〜9 で発火 | ✓ | — |
| 本番 / ドライラン切替 | ✓ | ✓ |
| モニター全画面 | ✓ | ✓ |
| 送信先・Cue・スケジュール編集 | — | ✓ |
| MIDI Learn | — | ✓（Cue タブ） |

現場本番中は **ショー** モードのまま運用し、設定変更が必要なときだけ **セットアップ** に切り替えます。

---

## 3.2 アプリ内での設定編集

**セットアップ** モードで以下を変更できます。

| 項目 | 編集場所 | 備考 |
|------|---------|------|
| 案件名・起動モード | **設定** | **保存** で書き込み |
| 送信先 | **送信先** | **適用** で即時保存 |
| Cue・アクション | **Cue** | **適用** で即時保存 |
| MIDI ノート | **Cue** → 編集 | **Learn** または数値入力 |
| OSC 引数 | **Cue** → 編集 | 行追加 UI（JSON 不要） |
| Cue 順序 | **Cue** 一覧 | ↑↓ または行ドラッグ |
| スケジュール | **スケジュール** | **適用** で即時保存 |
| 設定バックアップ | **設定** → **エクスポート** | |
| 設定差し替え | **設定** → **インポート** | 本番中は追加確認あり |

### 保存と反映

- CRUD の **適用**、**設定** の **保存**、**インポート** は即座にランタイムへ反映（再起動不要）
- **ファイルから再読込** — 外部 JSON 編集後に使用
- 保存時は `show.default.json.bak` が自動作成

### 本番モード中のロック

**LIVE** バナー表示中は、送信先・Cue・スケジュールの追加・編集・削除ボタンが無効になります。  
設定変更する場合は **ドライラン** に切り替えてから **セットアップ** モードで作業してください。

### 未保存の警告

- **設定** タブで未保存のまま **再読込** / **インポート** / タブ移動 → 確認ダイアログ

### 設定 JSON プレビュー

**開発者** プロファイル + **設定** タブ下部に表示。**現場** プロファイルでは非表示。

## 4. モードの使い方

### ドライラン（推奨：最初の確認）

- バナー: **DRY RUN — 送信しません**（黄色）
- 実際の OSC / UDP / MIDI Out は **送信しない**
- モニターに `[DRY_RUN]` / `[TRIGGER]` が表示される

### 本番

- バナー: **LIVE — 実送信しています**（赤色）
- **本番** ボタンクリック時に確認ダイアログ
- モニターに `[OSC]` / `[UDP]` / `[MIDI_OUT]` が表示される
- CRUD 編集がロックされる

切替: **発火** タブ上部の **本番** / **ドライラン** ボタン。

## 5. テスト手順

### 5.1 設定読み込みテスト

1. アプリ起動
2. モニターで以下を確認:
   - `[INFO] Loaded config: ...`
   - `[CONFIG] Validated config: ... targets, ... triggers, ... schedules`

### 5.2 セットアップウィザード

1. ヘッダー **セットアップウィザード** をクリック
2. ステップ 1 → **送信先** タブへ誘導
3. ステップ 2 → **Cue** タブへ誘導
4. ステップ 3 → **発火** タブでテスト発火

### 5.3 アプリ内 CRUD テスト

**前提:** ヘッダーを **セットアップ** に切替。

**送信先**

1. **送信先** → **追加** → OSC target → **適用**
2. **テスト（ドライラン）** でモニターにログ

**Cue**

1. **Cue** → **追加**
2. ラベル、OSC アドレス、引数（行追加 UI）を設定
3. **Learn** で MIDI ノートを取得 → **適用**
4. **発火** タブに Cue が表示され、ドライランで発火できること

**スケジュール**

1. **スケジュール** → **追加** → Cue を選択 → **適用**
2. **テスト発火** で紐づく Cue が発火すること

### 5.4 MIDI 入力テスト

**前提:** APC mini 等を接続。

1. **MIDI** タブ → **接続 OK** を確認
2. **再読み込み**
3. パッドを押す → モニターに `[MIDI] Note ON ...`
4. マッピング済みなら `[TRIGGER]` → `[DRY_RUN]` 等

**ノート番号の設定（UI）:**

1. **セットアップ** → **Cue** → 編集
2. **Learn（パッドを押す）** → パッドを押す → **適用**

### 5.5 画面 Cue 発火テスト

1. **ショー** モード、**ドライラン**
2. **発火** タブで Cue ボタンをクリック（またはキー `1`）
3. モニターに `[TRIGGER]` → `[DRY_RUN]` ログ
4. **最終 Cue** がダッシュボードに更新されること

### 5.6 キーボードショートカット

1. **ショー** モード
2. キー `1`〜`9` で先頭 9 Cue が順に発火すること
3. Cue ボタン右上にショートカット番号が表示されること

### 5.7 モニター全画面

1. **発火** タブ → モニターの **全画面** ボタン
2. ログが拡大表示されること
3. **閉じる** で元に戻ること

### 5.8 送信先単体テスト

1. **セットアップ** → **送信先** → **テスト（ドライラン）**
2. 問題なければ **本番** に切替 → **テスト（本番）**

### 5.9 複数 action 順次実行

1. actions が 3 件の Cue を **ドライラン** で発火
2. モニターで `Action 1/3` → `2/3` → `3/3` の順序を確認

### 5.10 スケジュールテスト

**手動:** **スケジュール** → **テスト発火**

**自動:** oneShot / daily を設定し、PC 時刻で自動発火 → `[SCHEDULE_FIRE]`

### 5.11 Cue 並び替え

1. **Cue** タブで ↑↓ またはドラッグで順序変更
2. **発火** タブのボタン順・キー 1〜9 の対応が変わること

### 5.12 本番モード CRUD ロック

1. **本番** に切替
2. **セットアップ** → 送信先/Cue/スケジュールの追加・編集・削除が無効
3. **ドライラン** に戻すと編集可能になること

## 6. 本番運用フロー

```
[開場 30 分前]
  PC 起動 → Signal Launcher 起動
  → ヘッダー: ショー / 現場
  → DRY RUN で全 Cue 1 回ずつ（キー 1〜9 も確認）
  → MIDI タブ: 接続 OK

[リハーサル]
  LIVE に切替（確認ダイアログ）→ 実機連動確認
  → 問題あれば DRY RUN に戻す
  → セットアップモードで修正 → 発火タブで再テスト

[本番]
  ショーモード + LIVE のまま運用
  → 発火タブを常時表示（2 カラム: Cue + モニター）
  → 必要ならモニター全画面をサブディスプレイに
  → APC mini + 画面ボタン + スケジュール

[終演後]
  セットアップ → 設定 → エクスポート（バックアップ）
  → logs/ を保存 → アプリ終了
```

## 7. ログの確認

| 出力先 | 内容 |
|--------|------|
| モニター（発火タブ） | リアルタイム。最新が上。全画面可 |
| `logs/YYYY-MM-DD.log` | 同日の全ログ |

主要カテゴリ: `[DRY_RUN]` `[OSC]` `[UDP]` `[MIDI_OUT]` `[TRIGGER]` `[MIDI]` `[CONFIG]` `[SCHEDULE_FIRE]` 等

## 8. トラブルシューティング

| 症状 | 確認・対処 |
|------|-----------|
| タブが少ない | **ショー** モード → **セットアップ** に切替 |
| MIDI 警告バナー | **MIDI** タブ → **接続 OK** になるまで USB・再読み込みを確認 |
| パッドを押しても反応なし | **Cue** タブで MIDI Learn、モニターの `[MIDI]` ログ |
| 設定を変更できない | **LIVE** 中はロック → **DRY RUN** + **セットアップ** |
| 本番に切り替わらない | 確認ダイアログで **キャンセル** していないか |
| OSC / UDP が届かない | **送信先** タブの host/port、ファイアウォール |
| 設定変更が反映されない | UI 保存は即反映。外部 JSON は **再読込** |
| Cue が削除できない | スケジュール参照を解除。または LIVE 中は不可 |
